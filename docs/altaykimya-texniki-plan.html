Layihə haqqında

AltayKimya avtomobil kimyası və təmizlik vasitələri üzrə onlayn mağazadır — kontaktsız yuma vasitələri, avtoşampunlar, ləkə təmizləyiciləri, mühərrik yuma, mum və qoruyucu vasitələr, salon/dəri təmizləyiciləri və aksesuarlar (brendlər: Grass, Express by Grass, Winso). Müştəri profili əsasən B2C (avtomobil sahibləri), həmçinin B2B potensialı (avtoyuma və detailing servisləri) mövcuddur.

Bu sənəd mövcud saytın Django + Next.js + PostgreSQL stack-i ilə yenidən qurulması üçün tam texniki planı təqdim edir.

1. Arxitektura yanaşması: Headless Commerce

Sayt headless (decoupled) arxitektura ilə qurulur: backend və frontend tam ayrıdır və aralarında REST API ilə əlaqə saxlayırlar.

┌──────────────────┐      HTTPS / JSON     ┌──────────────────┐
│   Next.js (BFF)   │ ◄───────────────────► │  Django + DRF     │
│   React frontend  │      REST API         │  (API + Admin)    │
└────────┬──────────┘                       └─────────┬─────────┘
         │                                            │
    ┌────▼─────┐                          ┌───────────▼───────────┐
    │  CDN /    │                          │  PostgreSQL 16         │
    │  Vercel   │                          │  Redis (cache + queue) │
    └───────────┘                          │  Celery (workers)      │
                                           │  Meilisearch (axtarış) │
                                           │  S3 / Object Storage   │
                                           └────────────────────────┘

Niyə headless? SEO və sürət üçün Next.js SSR/ISR imkanları, gələcəkdə mobil tətbiq əlavə etmək imkanı, frontend və backend komandasının müstəqil işləməsi, və admin üçün güclü Django Admin panelindən istifadə.

3. Backend strukturu (Django app-ları)

Modullar ayrı Django app-lara bölünür:

backend/
├── config/              # settings (base/dev/prod), urls, celery
├── apps/
│   ├── catalog/         # Category, Brand, Product, ProductVariant, Attribute
│   ├── inventory/       # Stock, anbar hərəkətləri

│   ├── orders/          # Order, OrderItem, status axını
│   ├── payments/        # Ödəniş provayderləri (Payriff, Azericard...)
│   ├── shipping/        # Çatdırılma zonaları, qiymət, kuryer
│   ├── accounts/        # User, address, B2B müştəri
│   ├── promotions/      # Endirim, kupon, kampaniya
│   ├── reviews/         # Məhsul rəyləri və reytinq

│   └── search/          # Meilisearch indeksləmə
└── common/              # ortaq utility, permissions, pagination

4. Verilənlər bazası sxemi (əsas modellər)

Product(id, category_id, brand_id, name_az/ru/en, slug, description,
        sku, is_active, created_at, meta_title, meta_description)
ProductVariant(id, product_id, name, sku, price, old_price,
        attributes(JSONB), barcode, weight, is_default)

Niyə Variant? Avtokimyada eyni məhsul müxtəlif həcmlərdə satılır (1L, 5L, 20L, kanistr). Hər həcm ayrı qiymət və stok deməkdir — buna görə ProductVariant modeli mütləqdir.

Sifariş və səbət (cart / orders)

Cart(id, user_id?, session_key?, created_at)        # qonaq + login
CartItem(id, cart_id, variant_id, quantity)

Order(id, number, user_id?, status, subtotal, discount, shipping_cost,
      total, payment_status, shipping_address(JSONB), phone,
      payment_method, created_at)
OrderItem(id, order_id, variant_id, product_name, price, quantity)
OrderStatusHistory(id, order_id, status, comment, created_at)

Sifariş statusları: pending → paid → processing → shipped → delivered → completed (+ cancelled, refunded).

İstifadəçi (accounts)

User(id, phone, email, first_name, last_name, is_b2b, ...)
Address(id, user_id, city, district, address_line, is_default)
B2BProfile(id, user_id, company_name, voen, discount_tier)   # B2B üçün

5. Əsas funksionallıqlar

Müştəri tərəfi (frontend)

  • Ana səhifə: banner slider, kateqoriyalar, populyar/yeni məhsullar, brendlər
  • Kataloq: kateqoriya səhifələri + filtrlər (brend, qiymət aralığı, həcm, reytinq) + sıralama
  • Məhsul səhifəsi: şəkil qalereyası, variant (həcm) seçimi, stok statusu, rəylər, oxşar məhsullar
  • Axtarış: anlıq (instant) axtarış, typo-tolerant, avtotamamlama
  • Səbət: qonaq səbəti (qeydiyyatsız), miqdar dəyişmə, kupon tətbiqi
  • Checkout: çatdırılma ünvanı, çatdırılma üsulu, ödəniş üsulu seçimi
  • Şəxsi kabinet: sifariş tarixçəsi, ünvanlar, seçilmişlər (wishlist)
  • B2B: topdan qiymət, VÖEN ilə qeydiyyat, faktura

Admin tərəfi (Django Admin — fərdiləşdirilmiş)

  • Məhsul / variant / stok idarəetməsi (kütləvi import — CSV/Excel)
  • Sifariş idarəetməsi və status dəyişmə
  • Endirim / kampaniya / kupon
  • Banner və CMS məzmun
  • Hesabatlar (satış, populyar məhsullar, stok azalması)

6. Ödəniş və çatdırılma (Azərbaycan üçün)

Ödəniş

  • Onlayn kart ödənişi: Payriff (PAŞA Bank) və ya Azericard / Kapital Bank gateway. Payriff developer-friendly API-ə malikdir — tövsiyə olunur.
  • Çatdırılma zamanı nağd/kart (cash on delivery) — Azərbaycanda hələ də geniş istifadə olunur.
  • Hissə-hissə ödəniş (Birkart/Tamkart/Bolkart) — gələcək mərhələ.
Təhlükəsizlik: Ödəniş üçün webhook + idempotency (təkrar bildirişlərə qarşı qorunma) mütləqdir. Ödəniş təsdiqi yalnız backend webhook ilə təsdiqlənməlidir — frontend redirect-ə güvənilmir.

Çatdırılma

  • Bakı daxili / regionlar üzrə zonalı qiymət (ShippingZone)
  • Pulsuz çatdırılma həddi (məs. 50 AZN-dən yuxarı)
  • Kuryer inteqrasiyası (gələcəkdə)

7. Axtarış strategiyası

  • Mərhələ 1 (MVP): PostgreSQL full-text search + pg_trgm (typo-ya dözümlü). Əlavə infrastruktur tələb etmir.
  • Mərhələ 2: Meilisearch — anlıq nəticələr, faceted filter, azərbaycan/rus dili dəstəyi, avtotamamlama. Məhsullar Celery ilə avtomatik indeksə sinxronlaşır.

8. SEO, performans və çoxdillilik

SEO (e-ticarət üçün kritik)

  • Next.js SSR/ISR ilə server-render edilən səhifələr
  • Hər məhsul/kateqoriya üçün dinamik meta, Open Graph, JSON-LD (Product, Offer, BreadcrumbList schema)
  • Avtomatik sitemap.xml və robots.txt
  • SEO-friendly slug-lar (/avtoshampun/grass-active-foam-5l)
  • Köhnə saytdan 301 redirect xəritəsi (mövcud URL reytinqini itirməmək üçün)

Performans

  • Şəkillər: next/image ilə avtomatik resize/WebP, lazy-load
  • ISR ilə kataloq səhifələrinin keşlənməsi, Redis keş
  • Lighthouse hədəfi: 90+ (mobil)

Çoxdillilik (i18n)

  • AZ (default) + RU — Azərbaycan bazarı üçün rus dili vacibdir, EN opsional
  • Backend: django-modeltranslation və ya ayrı dil sahələri
  • Frontend: next-intl

9. Təhlükəsizlik

  • HTTPS hər yerdə, HttpOnly + Secure cookie-lər
  • DRF rate limiting (login, checkout, axtarış)
  • CSRF/CORS düzgün konfiqurasiya
  • Ödənişdə server-side qiymət yoxlaması (frontend qiymətə güvənilmir)
  • Parol hashing (Argon2), admin üçün 2FA
  • Mütəmadi dependency yeniləməsi (Dependabot)
